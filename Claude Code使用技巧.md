
### 🛠️ Claude Code 常用操作指南

#### **一、核心交互与常用命令**

*   **启动与退出**：
    *   进入项目目录后，输入 `claude` 启动交互模式。
    *   输入 `exit` 或按 `Ctrl+C` 即可退出。
*   **三大核心模式**：通过 `Shift+Tab` 快捷键切换。
    *   **普通模式（默认）**：所有操作需你手动确认，最安全，适合新手。
    *   **自动接受模式**：AI 将自动执行所有操作，效率高，适合你完全信任它的简单场景。
    *   **计划模式（Plan Mode）**：AI 只制定详细计划而不修改代码。**强烈推荐**在开始复杂任务前，先用此模式对齐方案。
*   **常用对话内命令**：
    *   `/help`：查看所有可用命令。
    *   `/clear`：清空当前对话历史。当你觉得 AI “记混了”或要开启全新任务时使用。
    *   `/compact`：压缩冗长的对话上下文，帮助 AI 聚焦当前任务，还能节省 Token。
    *   `/init`：让 AI 扫描整个项目并生成 `CLAUDE.md` 文件。这是让 AI 理解你项目的关键一步。
    *   `/doctor`：检查 Claude Code 的运行环境是否正常，排查问题时用。
*   **模式切换**：在对话中直接输入 `/model` 可以选择切换 sonnet、opus 或 haiku 模型。对应你配置的 DeepSeek 模型即为 V4-Pro 或 V4-Flash。

#### **二、进阶工作流与技巧**

掌握这些技巧，能让 AI 更好地配合你的开发节奏：

*   **用好项目“记忆”文件 `CLAUDE.md`**：执行 `/init` 后，项目根目录会生成一个 `.claude/CLAUDE.md` 文件。你可以随时手动编辑它，加入项目命名规范、技术栈、核心逻辑等说明。这样，每次启动 Claude Code 时它都会自动读取，无需你再重复解释。
*   **高效的指令技巧**：
    *   **上下文引用**：使用 `@文件名` 可以快速让 AI 关注特定文件，比如 `解释 @src/utils/date.ts`。
    *   **终端指令**：在对话中直接以 `!` 开头，就能执行终端命令，例如 `! npm test`。
    *   **自定义命令**：在 `.claude/commands/` 目录下创建 `.md` 文件，就能生成专属的斜杠命令。比如，创建一个 `review.md`，写入“检查我最近修改的代码是否符合规范”，之后直接输入 `/review` 就能复用这个检查流程。
*   **版本控制集成**：
    *   直接说 `commit my changes`，AI 会自动生成 commit message 并提交。
    *   它还能帮你创建分支、查看修改历史或解决合并冲突，例如 `create a branch called feature/new-api`。
*   **与 VS Code / IDE 集成**：如果你使用 VS Code，可以安装官方扩展，并在 Claude Code 会话中输入 `/ide` 连接 IDE。之后，AI 能直接感知你在编辑器里选中的代码，并通过可视化 Diff 视图展示代码修改，体验远超纯终端。
*   **自动化与脚本**：使用 `claude -p "你的任务"` 命令，可以让 AI 以非交互模式完成一次性任务，适合集成到脚本或 CI/CD 流程中。

#### **三、特别提醒**

由于你使用的是 DeepSeek API，有两个特有的小情况需要留意：
*   **纯文本模型**：你当前配置的 `deepseek-v4-pro` 是纯文本模型，**不支持**官方 Claude Code 的图片分析功能。
*   **稳定性**：像之前聊到的，调用第三方 API 偶尔会遇到超时，这通常与模型本身或网络有关，和你的配置关系不大。

总的来说，Claude Code 的上手路径很清晰：先用 `/init` 让 AI 熟悉项目，接着用 `Shift+Tab`进入计划模式讨论方案，最后回到普通模式执行代码修改，并善用 `/compact` 和 `/clear` 管理会话。
