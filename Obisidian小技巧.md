Obsidian 的 Callout（标注/提示框) 语法。它基于 Markdown 的块引用（`>`）扩展而来，用于创建带图标和背景色的高亮信息块。

### 基础语法结构

```
> [!类型] 自定义标题
> 这是提示框的内容。
> 支持**加粗**、`代码`等 Markdown 语法。
```

- **`>`**：Markdown 块引用符号，每一行内容前都需要。
    
- **`[!类型]`**：定义 Callout 的类型，如 `note`, `tip`, `warning`等，决定了颜色和图标。
    
- **自定义标题**：`]`后面的文字会显示为标题。如果省略，则默认显示类型名（如 "Note"）。
    

### 常用类型速查

Obsidian 内置了多种类型，**类型名不区分大小写**（`[!NOTE]`和 `[!note]`效果一样）。

| 类型                   | 别名                | 典型用途      |
| -------------------- | ----------------- | --------- |
| `note`               | -                 | 普通备注、说明   |
| `info`               | -                 | 信息提示      |
| `tip`/ `hint`        | `important`       | 技巧、建议     |
| `warning`/ `caution` | `attention`       | 警告、注意     |
| `danger`/ `error`    | -                 | 危险操作、严重错误 |
| `success`/ `done`    | `check`           | 成功、已完成    |
| `todo`               | -                 | 待办事项      |
| `abstract`           | `summary`, `tldr` | 摘要、总结     |
| `question`/ `faq`    | `help`            | 问题、帮助     |
| `bug`                | -                 | Bug 报告    |
| `example`            | -                 | 代码/用法示例   |
| `quote`/ `cite`      | -                 | 引用他人话语    |

### 高级用法

1. **折叠与展开**
    
    在类型后加 `+`（默认展开）或 `-`（默认折叠）来控制状态。
    
    ```
    > [!FAQ]- 点击展开常见问题
    > 这里是隐藏的答案。
    ```
    
2. **嵌套使用**
    
    在内层内容前多加一个 `>`即可实现嵌套。
    
    ```
    > [!question] 外层问题
    > > [!tip] 内层提示
    > > 这里是嵌套的内容。
    ```
    
3. **仅标题（无内容）**
    
    如果只写标题，不写内容行，会渲染成一个只有标题的块。
    
    ```
    > [!NOTE] 这是一个只有标题的 Callout
    ```
    

### 示例：你提到的 NOTE

```
> [!NOTE] 这是标题
> 这是具体的说明内容。
> 第二行内容。
```

渲染效果：

> [!NOTE] 这是标题
> 
> 这是具体的说明内容。
> 
> 第二行内容。

### 注意事项

- **换行**：Obsidian 的实时预览模式通常会自动补全换行前的 `>`，但为了兼容性，手动换行时最好在每行开头都加上 `>`。
